# 本地贴吧系统 v1.1 开发规划 (学生大作业版)

## 核心目标
在 v1 (纯文本发帖、WebSocket聊天、单机Tomcat) 的基础上，增加高性价比、易于答辩展示的亮点功能，重点体现对数据库复杂查询、异步交互、文件处理的掌握。

## 模块一：核心体验与查询能力展示 (必做)

### 1. 全局分页 (Pagination)
*   **应用场景**：贴吧主题帖列表 (`forum_thread`)、帖子详情的楼层列表 (`forum_post`)。
*   **技术价值**：展示 `LIMIT offset, size` 的使用，以及前端分页组件逻辑封装。
*   **数据结构**：无需新增表。
*   **核心 SQL**：
    ```sql
    SELECT * FROM forum_thread WHERE board_id = ? ORDER BY last_reply_time DESC LIMIT ?, ?;
    SELECT count(*) FROM forum_thread WHERE board_id = ?;
    ```

### 2. 浏览量与回复数统计
*   **应用场景**：帖子列表展示 `view_count` 和 `reply_count`。
*   **技术价值**：展示对数据的更新和同步处理。
*   **核心逻辑**：
    *   用户访问帖子详情页时，异步或同步执行 `UPDATE forum_thread SET view_count = view_count + 1 WHERE id = ?`。
    *   用户发表回复时，触发 `UPDATE forum_thread SET reply_count = reply_count + 1, last_reply_time = NOW() WHERE id = ?`。

## 模块二：个人中心与多表联查 (高分项)

### 1. 个人主页与头像上传
*   **应用场景**：用户登录后可修改昵称、简介，并上传本地图片作为头像。
*   **技术价值**：展示 **文件上传** 处理、Tomcat 静态资源映射、以及基础的 IO 操作。
*   **核心逻辑**：
    *   前端使用 `<input type="file">` 和 `FormData` 提交。
    *   后端使用 Spring MVC 的 `MultipartFile` 接收图片，保存至 Tomcat 配置的外部目录（如 `data/uploads/`）。
    *   更新 `forum_user_profile` 表的 `avatar` 字段为图片 URL (例如 `/uploads/xxx.jpg`)。

### 2. 我的足迹 (多表联查)
*   **应用场景**：个人中心展示“我发布的帖子”、“我回复的帖子”。
*   **技术价值**：展示复杂的 `JOIN` 查询能力。
*   **核心 SQL** (我发布的帖子)：
    ```sql
    SELECT t.id, t.title, t.created_at, b.name as board_name
    FROM forum_thread t
    LEFT JOIN forum_board b ON t.board_id = b.id
    WHERE t.user_id = ?
    ORDER BY t.created_at DESC;
    ```

## 模块三：异步交互与页面优化 (亮点项)

### 1. AJAX 异步点赞
*   **应用场景**：用户对帖子或楼层点赞，数字无刷新 +1。
*   **技术价值**：展示 `AJAX / Fetch API` 异步请求、JSON 数据交互。
*   **数据模型变更**：
    *   `forum_thread` 表新增 `like_count` 字段 (INT, 默认 0)。
    *   `forum_post` 表新增 `like_count` 字段 (INT, 默认 0)。
    *   *(可选进阶)* 新增点赞记录表 `forum_like_log (user_id, target_id, target_type)` 防止重复点赞。

### 2. 轻量级 Markdown/富文本发帖
*   **应用场景**：发帖时支持加粗、插入网络图片。
*   **技术价值**：纯前端组件集成，提升项目整体“卖相”。
*   **核心逻辑**：
    *   引入 WangEditor (富文本) 或 Editor.md (Markdown)。
    *   后端保持接收和存储字符串 (HTML/Markdown原文)，前端渲染时解析。

## 模块四：站内搜索与排行榜 (进阶项)

### 1. 标题模糊搜索
*   **应用场景**：导航栏搜索框，输入关键字查询全站帖子。
*   **技术价值**：展示 `LIKE` 模糊查询。
*   **核心 SQL**：
    ```sql
    SELECT t.*, u.nickname FROM forum_thread t LEFT JOIN forum_user_profile u ON t.user_id = u.user_id WHERE t.title LIKE CONCAT('%', ?, '%') ORDER BY t.created_at DESC;
    ```

### 2. 首页热门榜单
*   **应用场景**：首页侧边栏展示当前回复最多的 Top 10 帖子。
*   **技术价值**：展示聚合排序和 `LIMIT` 截取。
*   **核心 SQL**：
    ```sql
    SELECT id, title, reply_count FROM forum_thread ORDER BY reply_count DESC, view_count DESC LIMIT 10;
    ```
