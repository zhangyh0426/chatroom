# 模块：个人中心与头像上传 (技术设计文档)

## 1. 业务目标
为学生大作业提供一个视觉效果好且能展示后端文件处理能力的模块。用户登录后可以进入“我的主页”，修改个人昵称、签名（简介），并支持本地图片作为头像上传。

## 2. 核心技术考点 (答辩加分项)
1. **HTML/FormData 表单提交**：包含 `<input type="file">` 的表单，`enctype="multipart/form-data"` 的处理。
2. **Spring MVC 文件上传 (`MultipartFile`)**：接收并解析前端传来的二进制图片流。
3. **本地文件系统 IO 操作**：将接收到的文件写入服务器（Tomcat）宿主机指定的绝对路径下（如 `F:\zhao\chatroom\data\uploads`）。
4. **Tomcat 虚拟目录映射 (重要)**：解决本地上传的图片如何在前端通过 URL (如 `http://localhost:8080/uploads/avatar.jpg`) 直接访问的问题。
5. **数据库 UPDATE 操作**：更新 `forum_user_profile` 表中的 `avatar` 和 `bio` 字段。

## 3. 数据模型设计
依赖于 v1 基础规划中的 `forum_user_profile` 表：

```sql
CREATE TABLE `forum_user_profile` (
  `user_id` BIGINT PRIMARY KEY COMMENT '关联 forum_user_account.id',
  `nickname` VARCHAR(50) NOT NULL COMMENT '用户昵称',
  `avatar` VARCHAR(255) DEFAULT '/static/images/default_avatar.png' COMMENT '头像URL路径',
  `bio` VARCHAR(255) DEFAULT '这个人很懒，什么都没留下' COMMENT '个人简介/签名',
  `last_login_ip` VARCHAR(50),
  `last_login_time` DATETIME
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 4. 后端接口设计 (Spring MVC)

### 4.1 页面路由
* **`GET /user/profile`**：渲染个人中心页面 (`profile.jsp`)。
  * 控制器：`UserController.showProfile(HttpSession session, Model model)`
  * 逻辑：从 Session 获取当前登录的 `user_id`，查询 `forum_user_profile` 并放入 `Model`。

### 4.2 头像上传与资料更新接口
* **`POST /api/user/profile/update`**：处理表单提交。
  * **参数**：
    * `nickname` (String): 新昵称
    * `bio` (String): 新签名
    * `avatarFile` (`MultipartFile`): 上传的头像文件 (非必填，若不传则不更新头像)
  * **逻辑伪代码**：
    ```java
    @PostMapping("/api/user/profile/update")
    public String updateProfile(
            @RequestParam("nickname") String nickname,
            @RequestParam("bio") String bio,
            @RequestParam(value = "avatarFile", required = false) MultipartFile avatarFile,
            HttpSession session, 
            RedirectAttributes redirectAttrs) {
            
        Long userId = (Long) session.getAttribute("USER_ID");
        if (userId == null) return "redirect:/login";

        String avatarUrl = null;

        // 1. 处理文件上传 (核心考点)
        if (avatarFile != null && !avatarFile.isEmpty()) {
            // 获取原始文件名并生成唯一新文件名 (防止覆盖)
            String originalFilename = avatarFile.getOriginalFilename();
            String extension = originalFilename.substring(originalFilename.lastIndexOf("."));
            String newFilename = UUID.randomUUID().toString() + extension;
            
            // 物理保存路径 (需在外置配置文件中配置，例如: F:/zhao/chatroom/data/uploads/)
            String uploadDirPath = "F:/zhao/chatroom/data/uploads/"; 
            File dest = new File(uploadDirPath + newFilename);
            
            try {
                // 如果目录不存在则创建
                if (!dest.getParentFile().exists()) dest.getParentFile().mkdirs();
                // 写入磁盘
                avatarFile.transferTo(dest);
                // 生成前端访问的 URL (假设 Tomcat 映射了 /uploads 虚拟路径)
                avatarUrl = "/uploads/" + newFilename; 
            } catch (IOException e) {
                redirectAttrs.addFlashAttribute("error", "头像上传失败！");
                return "redirect:/user/profile";
            }
        }

        // 2. 更新数据库
        // 调用 UserService.updateProfile(userId, nickname, bio, avatarUrl);
        // SQL: UPDATE forum_user_profile SET nickname = ?, bio = ? [, avatar = ?] WHERE user_id = ?

        // UserProfile profile = userService.getProfile(userId);
        // session.setAttribute("USER_PROFILE", profile);

        redirectAttrs.addFlashAttribute("success", "资料更新成功！");
        return "redirect:/user/profile";
    }
    ```

## 5. Tomcat 虚拟路径映射配置 (关键步骤)

为了让前端能够通过形如 `<img src="/uploads/xxxxx.jpg">` 访问到刚才保存在 `F:/zhao/chatroom/data/uploads/` 目录下的图片，必须在 Tomcat 中配置虚拟目录。

**方法：修改 Tomcat 的 `server.xml`**
在 `<Host name="localhost" ...>` 标签内部添加：

```xml
<Context docBase="F:\zhao\chatroom\data\uploads" path="/uploads" reloadable="true" />
```
* **解释**：当浏览器请求 `http://localhost:8080/uploads/xxxx.jpg` 时，Tomcat 会自动去 `F:\zhao\chatroom\data\uploads\xxxx.jpg` 寻找该文件。

## 6. 前端 JSP 页面设计思路 (`profile.jsp`)

```html
<!-- 表单必须加 enctype="multipart/form-data" -->
<form action="${pageContext.request.contextPath}/api/user/profile/update" method="POST" enctype="multipart/form-data">
    
    <!-- 提示信息 -->
    <c:if test="${not empty success}"><div class="alert alert-success">${success}</div></c:if>
    <c:if test="${not empty error}"><div class="alert alert-danger">${error}</div></c:if>

    <!-- 头像展示与上传 -->
    <div class="form-group">
        <label>当前头像</label>
        <img src="${userProfile.avatar}" alt="Avatar" width="100" height="100" class="img-circle">
        <br>
        <label for="avatarFile">更换头像</label>
        <input type="file" id="avatarFile" name="avatarFile" accept="image/png, image/jpeg, image/gif">
    </div>

    <!-- 昵称 -->
    <div class="form-group">
        <label for="nickname">昵称</label>
        <input type="text" class="form-control" id="nickname" name="nickname" value="${userProfile.nickname}" required>
    </div>

    <!-- 个人简介 -->
    <div class="form-group">
        <label for="bio">个人简介</label>
        <textarea class="form-control" id="bio" name="bio" rows="3">${userProfile.bio}</textarea>
    </div>

    <button type="submit" class="btn btn-primary">保存修改</button>
</form>
```
